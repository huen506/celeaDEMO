# CLAUDE.md — Celēa 專案工作規範

這份文件定義 Claude Code 在 Celēa 專案裡的固定工作習慣。每次開工都必須遵守。

---

## 一、開工前確認

每次啟動後，依序執行：

1. 確認 MCP 連線：輸入 `/mcp`，確認 `unityMCP · ✔ connected`
   - 若未連線，提示作者先在 Unity 執行 Window → MCP For Unity → Start Server
2. **讀取藍圖**：`cat docs/Celēa_工程實作藍圖_v1.md`
   - 藍圖包含完整接口、事件表、資料夾結構、Singleton 分級——先讀藍圖再動手，不要直接撈 CS

---

## 二、操作物件的習慣（最重要）

**每次收到操作指令之前，先呼叫 `editor_selection` 讀取目前在 Unity 編輯器裡選取的物件。**

流程：
1. 作者在 Unity 點擊一個物件
2. 作者下達指令（例如「幫我掛上 DialogueManager」）
3. Claude Code 呼叫 `editor_selection` 確認選取的物件是什麼
4. 把選取的物件當作操作對象，直接執行

**如果沒有選取任何物件：** 不要自己猜，詢問作者「請問要對哪個物件操作？」

**如果選取的物件跟指令明顯不符：** 說明目前選取的是什麼，詢問作者是否確認。

## 五、執行規則

- **規格外不擅自新增：** 沒有明確指示的功能不自行新增，不確定就停下來詢問
- **每步確認再繼續：** 每個操作完成後確認 Unity 無編譯錯誤，再進下一步
- **美術端未定義的數值：** 一律用佔位常數，不自行決定（例如暗化比例、縮放比例）
- **發現規格落差：** 立即說明，不單方面修改設計邏輯
- **操作完記得存場景：** Claude Code 對場景的修改不會自動存檔，操作完提醒作者按 Ctrl+S

## 六、已知注意事項

- Port 8090 不能被其他程式佔用
- Claude Code 必須從專案資料夾啟動（.mcp.json 在專案根目錄）
- 修改 CS 檔案後需等待 Unity 編譯完成再繼續操作
- Inspector 的物件引用連線只能靠程式碼或手動拖拉，MCP 無法模擬拖拉操作
